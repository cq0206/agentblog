# 工具调用与工作流

## 为什么要工具调用

纯对话模型擅长生成，但不擅长直接操作真实环境。  
引入工具后，Agent 才能读取文件、执行命令、访问外部系统并返回可验证结果。

## 最小工作流

1. 解析任务：明确输入、输出、约束
2. 规划步骤：决定调用哪些工具
3. 执行工具：逐步运行并记录结果
4. 校验输出：检查是否满足验收标准
5. 生成结论：给出结果和下一步建议

## 常见失败点

- 一次调用做太多事，导致定位困难
- 没有中间状态记录，失败后无法复盘
- 未定义验收标准，结果看起来“像对的”

## 改进策略

- 拆分为可观测的小步骤
- 每一步输出结构化结果
- 失败时优先返回原因和可执行修复动作
